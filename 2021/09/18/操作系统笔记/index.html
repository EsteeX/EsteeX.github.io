Chapter 1

操作系统的功能

计算机系统：硬件（cpu，内存、输入输出设备）、操作系统（相当于政府，提供环境）、应用程序（字处理程序、编译器）、用户

操作系统——资源分配器 resource allocator 控制程序 control program

计算机上的程序：1. 内核（操作系统，一直运行） 2. 系统程序（系统运行有关，不是内核的一部分） 3. 应用程序

移动的操作系统 包括 内核 和 中间件

引导程序

开机时：引导程序——位于计算机固件，如只读内存（ROM），电可擦可编程只读内存（EEPROM）

初始化各个组件（CPU寄存器、设备控制器、内存内容），加载操作系统（定位操作系统内核，加载到内存），开始执行系统。

系统程序也 被加入内存，成为系统进程（system process），生命周期 同内核。

对于UNIX 首个系统进程：init，启动许多其他系统的后台程序

中断

事件发生，中断cpu进程

硬件 随时 通过 系统总线 发送信号 到cpu，触发中断。

软件 通过系统调用（system call），触发中断。

cpu被中断

有不同的中断机制，但是都有共同点：

  1. 中断将控制转移到合适的中断服务程序。

调用一个 通用程序，检查中断信息。，该程序再调用特定的中断处理程序。

  1. 中断处理应该快捷

    通过中断处理程序的指针 提高速度。通过指针表间接调用中断处理程序。地址组成的数组：中断向量 intertupt vector

    将返回地址保存在系统堆栈上

存储结构

cpu只能从内存中加载指令，因此执行程序必须位于内存。

内存通常为动态随机访问内存（DRAM）

还有ROM，不可修改， 只能将静态程序（如引导程序放入其中）

EEPROM可以修改，但不能经常修改。保存大多数静态程序（手机工厂安装的软件）

程序和数据 不能 永远驻留在内存中

  • 内存太少，不能永久保存所需要的程序和数据
  • 内存是易失性（volatile）存储设备，掉电时会失去所有内容

外存

secondary storage

扩充内存

永久存储大量数据

Ep： 磁盘、硬盘

I/O结构

设备驱动程序 device driver

开始I/O时，设备驱动程序加载 设备控制器 的 适当寄存器， 设备控制器读取寄存器的内容，决定采取什么操作（ep，从键盘读入一个字符

计算机系统体系结构

单处理器系统

一个通用cpu。（虽然有的环境下有其他专用处理器，但是只有一个通用cpu，所以也只能算单处理器系统）

多处理器系统

Multiprocessing system也称并行系统

多个cpu共享计算机总线、时钟、内存、外设

优点：

  • 增加吞吐量

    N个处理器的加速小于N。因为合作会有额外开销

  • 规模经济

    N处理器系统的价格<N*单处理器系统

    因为可以共享计算机总线、时钟、内存、外设

  • 增加可靠性

    单个处理器失灵不会使整个系统停止，只是会使他变慢

    适度退化 graceful degradation

    根据剩余有效硬件的级别按比例继续提供服务的能力

    容错 fault tolerant

    超过适度退化。需要一定的机制来对故障进行检查

两种类型：

非对称处理 asymmetric multiprocessing

主处理器调用从处理器 安排工作

对称处理 symmetric multiprocessing （SMP）

每个处理器都参与完成操作系统的所有任务。

有自己的寄存器集，缓存，但共享物理内存 ep AIX

优点：多进程可以同时进行，且并不会明显地影响性能。

UMA & NUMA

均匀内存访问（UMA）—-多处理—>非均匀内存访问

操作系统通过资源管理可以改善NUMA问题

多核 & 单核

计算核 computing core

多核 multicore

集成多个计算核到单个芯片

多核系统为多处理系统，但多处理系统不一定是多核的

操作系统的结构

具有多道程序能力

多道程序设计 multiprogramming

作业池 job pool：

操作系统在内存中保存多个任务，主存不够用，所以这些作业首先保存在磁盘的作业池上

内存的作业集 属于 作业池的作业集

作业调度 job scheduling

只要有一个任务可以执行，CPU就不会空闲

分时系统

time sharing

还是通过切换作业来执行多个作业，但是切换频率很高，用户可以在程序运行时与其交互。

操作系统的执行

中断驱动 interrupt driven

操作系统 等待时间发生

事件：中断/陷阱trap（或异常 exception）

陷阱：软件生成中的中断（出错 如除数为0/无效存储访问 / 用户程序的特定请求）

操作系统 必须保证 错误程序不会造成其他程序的错误执行

->双重模式 多重模式

需要硬件支持

区分操作系统代码和用户系统代码的执行

至少两种单独运行模式 用户模式 内核模式kernel mode

通过一个模式位 mode bit 表示当前模式 0——内核 1——用户

特权指令 privileged instruction 硬件只有在内核模式下才允许执行特权指令

切换到用户模式的指令 属于 特权指令

模式可以超过两个：多个模式位

支持虚拟化（virtualization） 多了一种模式——表示虚拟机管理器（Virtual Machine Manager VMM)是否正在控制系统 特权多于用户模式，少于内核模式 可以改变CPU状态

除了模式 ， 还可以

设置四种特权级别 privilege level

系统调用：陷入中断向量的某个指定位置

方式：一般由通用trap指令完成 MIPS系统由专用syscall指令来完成系统调用

定时器

区分定时器和时钟（定时器用来设置产生中断，时钟用来实现定时器）

计数器不仅用于计数时钟，也可以计数定时器

目的：控制CPU，防止用户程序陷入死循环

可以设置为在指定周期后中断计算机

指定周期：固定/可变 （可变定时器 variable timer——通过一个固定速率的时钟和计数器来实现。操作系统控制计数器，时钟存在精度。时钟每走一个步长，计数器-1，值为0时产生中断。

防止用户程序运行过长方法：设置程序运行7分钟：设置定时器每秒产生一次中断，计数器设为420，每产生一次中断计数器减1.当计数器值为负时，操作系统终止程序执行。

进程管理

被CPU执行的程序——进程

进程可以通过系统调用 创建子进程 以并发执行。

程序本身不是进程，是个被动实体（passive entity）

进程是主动实体（active entuty）

单线进程

有一个程序计数器（program counter），指定下一个所要执行的指令。

在任何时候，每个进程最多只能执行一条指令

多线程进程 有多个pc

进程分为 操作系统进程和用户进程。 所有这些进程都会并发执行（ep 值单CPU上采用多路复用）

内存管理

为CPU和I/O设备共享

CPU唯一能直接寻址访问的存储器。（所以指令必须在内存中）

CPU若要处理磁盘数据，需要 CPU产生I/O调用将数据传到内存。

操作系统负责：

  • 记录内存哪部分在使用&被谁使用
  • 决定哪些进程 回调入/调出内存
  • 分配、释放内存空间

存储管理

对存储设备物理属性进行抽象 定义逻辑存储单元——文件

操作系统通过 存储设备 来访问文件

文件管理系统

操作系统的组件之一

文件内容是程序和数据。

文件可组织成目录

操作系统负责：

  • 创建和删除文件
  • 创建和删除目录
  • 提供文件和目录 的操作原语
  • 映射文件到外存
  • 备份文件到稳定的存储介质

大容量存储器管理

硬盘（外存），程序执行师才被调入内存

硬盘作为处理的起源和终点

操作系统负责：

  • 空闲空间管理
  • 存储空间分配
  • 硬盘调度

三级存储 tertiary storage

比外存速度更慢，价格更低

高速缓存

caching

原理：信息通常保存在一个存储系统（如内存），使用时，它会被临时复制到更快存储系统，即高速缓存

内存可以看为外存的高速缓存

高速缓存一致性 cache coherence（硬件问题）——对于多处理器环境，每个CPU 有自己的内部寄存器 和 本地高速缓存。数据的拷贝可能出现在多个缓存上。需要确保 一个高速缓存的A值更新，应马上反应到所有其他A所在的高速缓存。
